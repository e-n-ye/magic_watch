# 视觉资产工作流

本文档定义 Magic Watch 当前阶段的图片生成与图片分析工作流。

## 目标

当前我们已经有了主页环交互骨架，下一步视觉不应该直接写死在代码里盲试，而应该先形成一套可复用的视觉预研流程：

- 用图片理解能力拆解参考图
- 用图片生成能力快速产出表盘 / 快捷页视觉方向
- 把结论回写到项目文档和代码，而不是只留在聊天里

## 当前约定

- 项目内长期保留的是：提示词模板、脚本、结构化分析结论
- 生成出的临时图片和 API 返回默认放在 `.agents/generated/`
- API key 不进 git，本地写入 `.agents/config/right_code.env`

## 技能入口

项目级技能位置：

- `.agents/skills/watch-visual-workflow/`

核心内容：

- `SKILL.md`：何时使用、如何推进
- `scripts/right_code_generate_image.py`：生成图片
- `scripts/right_code_chat.py`：分析图片 / 参考图
- `references/`：API 速记和提示词模式
- `assets/prompts/`：可复用提示词模板

## 本项目推荐使用方式

### 1. 先分析，再生成

当你手里已经有小米手表、InfiniTime、其他可穿戴产品截图时：

1. 先用 `right_code_chat.py` 做结构拆解
2. 提炼出：
   - 层级
   - 主视觉焦点
   - 信息密度
   - 可压缩到 v0 的部分
3. 再基于这些结论写生成提示词

### 2. 一次只打一个表面

不要一次生成“整套系统界面”。

当前最适合拆开的目标是：

- 表盘
- 主页环单个快捷页
- 快捷设置页
- Launcher 页

### 3. 先服务结构，再服务美术

当前阶段生成图的第一目的不是做最终视觉稿，而是帮助我们回答：

- 这一页最该突出什么
- 卡片层次怎么排
- 圆形表盘安全区怎么留
- 哪些内容应该是图片氛围，哪些应该是数据驱动

## 推荐顺序

1. 主页环 4 个快捷页统一结构
2. 先给 `Weather / Steps / Sleep` 这一页生成第一批视觉参考
3. 用结果反推：
   - 背景层
   - 主卡片层
   - 指标卡层
   - 文本层
4. 再决定是否给表盘主页面引入背景图或动态资产

## 边界

- 当前工作流是预研工具，不代表最终量产资源链路
- 当前生成图主要用于方向验证，不直接当最终 UI 成品
- 真正进入固件时，要继续收缩为：
  - 可解释的布局
  - 可替换的图片资源
  - 可数据驱动更新的内容层
