# T-Watch S3 Plus 参考板核验计划

日期：2026-05-21

## 目标

本轮使用用户现有的 LILYGO T-Watch S3 Plus 作为第一块参考验证板，快速核验真实手表硬件对当前模拟器架构假设的影响。

本轮不是最终硬件选型，不确认最终主控、屏幕、触摸、IMU 或 PMIC。T-Watch S3 Plus 只作为参考板，用来降低未知并为后续分立器件选型提供对照。

## 已知限制

用户已反馈：当前这块 T-Watch 中的 BMA423 计步中断疑似存在硬件问题，暂时无法实现计步。

因此本轮做出以下调整：

- 不把计步中断作为验收项。
- 不把 Steps / Pedometer app 接入真实计步作为本轮目标。
- 不用 BMA423 计步能力判断该板是否适合作为参考验证板。
- IMU 核验只关注基础读数、姿态变化、普通中断能力和 `RaiseToWake` 替代验证路径。
- 如果 BMA423 中断路径继续异常，应记录为“当前板级限制”，不要立即推导为所有 IMU 方案不可行。

## 本轮不做什么

- 不移植完整模拟器 UI。
- 不重写 `HAL::Device`。
- 不建立正式 BSP。
- 不做分立器件飞线验证。
- 不决定最终主控是否为 ESP32-S3。
- 不决定最终屏幕是否为 240 x 240。
- 不决定最终触摸控制器是否为 FT6336U。
- 不决定最终 IMU 是否为 BMA423。
- 不决定最终 PMIC 是否为 AXP2101。

## 核验问题

本轮只回答以下问题：

1. T-Watch S3 Plus 能否跑起最小 LVGL 画面。
2. 240 x 240 真实手表屏幕会如何压缩当前 240 x 296 模拟器 UI 假设。
3. FT6336U 触摸能否稳定提供点击、拖动和滑动基础事件。
4. 物理按键能否作为 `CrownPress` / wake 输入的替代路径。
5. BMA423 是否至少能提供基础姿态数据或非计步类中断。
6. AXP2101 是否能读取电池、充电、外部供电和基础电源状态。
7. screen off / wake 后哪些 UI / 运行时状态可以保持，哪些必须重建。
8. 当前模拟器中的 `TapToWake`、`RaiseToWake`、AOD / 息屏显示假设是否需要拆分或降级。

## 实验分组

### A. 最小系统与显示实验

目标：

- 确认可烧录、可运行、可输出日志。
- 点亮 240 x 240 屏幕。
- 运行最小 LVGL 画面。

验收：

- 串口日志可见。
- 屏幕能稳定显示一个基础 LVGL 页面。
- 能记录屏幕分辨率、旋转方向、颜色格式和刷新方式。

不要求：

- 显示当前 Magic Watch 全量 UI。
- 适配模拟器页面。
- 验证性能上限。

### B. 240 x 240 UI 压力观察

目标：

- 用真实屏幕观察当前模拟器 UI 假设的主要风险。
- 判断 240 x 296 模拟器方向是否需要新增 240 x 240 压力测试视角。

观察项：

- 表盘主要信息是否容易拥挤。
- Launcher 图标墙或列表是否需要不同密度。
- Settings 列表是否仍可读。
- screen-off fallback 样式是否能在 240 x 240 上成立。

验收：

- 产出一份观察记录。
- 明确哪些问题是 T-Watch 的 240 x 240 屏幕导致，哪些可能是所有小屏手表的共性。

不要求：

- 立即修改模拟器 UI。
- 立即新增 240 x 240 模拟器配置。

### C. 触摸输入实验

目标：

- 确认 FT6336U 基础触摸路径可用。
- 判断触摸事件是否足以映射到当前 `TouchSample` 思路。

验收：

- 能读取按下、移动、释放或等价事件。
- 能区分点击和拖动。
- 能记录坐标范围、坐标方向和边缘触摸可靠性。
- 能初步判断边缘返回手势是否有硬件层限制。

待核验：

- FT6336U 是否支持低功耗触摸唤醒。
- 触摸中断在 screen off 或 light sleep 状态下是否仍可用。

### D. 物理输入 / 唤醒替代实验

目标：

- 确认物理按键能否作为 `CrownPress` 和 wake 输入的替代。

验收：

- 至少一个物理按键能产生稳定事件。
- 能区分短按和长按，或明确当前只能验证短按。
- 能记录按键在 screen off / sleep 状态下是否还能唤醒。

不要求：

- 验证真实表冠旋转。
- 验证 `CrownSample::RotateCW / RotateCCW`。

### E. BMA423 姿态 / 中断实验

目标：

- 在计步中断疑似异常的前提下，评估 BMA423 还能提供哪些可用能力。

验收：

- 能读取基础加速度数据。
- 能观察手腕翻转、抬起、放下等姿态变化的原始数据差异。
- 如果普通运动中断或方向中断可用，记录中断类型、触发条件和稳定性。
- 如果所有中断路径异常，记录为当前板级风险。

明确不验收：

- 计步中断。
- 真实计步算法。
- Steps app 真实数据接入。

对 `RaiseToWake` 的处理：

- 如果 BMA423 可用中断足够稳定，可进入 `RaiseToWake` 替代验证。
- 如果中断不可用，只用运行态姿态数据验证产品语义，不声称已验证低功耗翻腕亮屏。

### F. AXP2101 电源 / 电池实验

目标：

- 确认电源管理和电池状态的可观察性。

验收：

- 能读取电池电压或等价电量信息。
- 能读取充电状态或外部供电状态。
- 能控制或观察屏幕背光 / 显示电源状态。
- 能记录 screen off、背光关闭、light sleep、deep sleep 之间的差异。

待核验：

- AXP2101 是否能支持必要的外设电源域管理。
- 进入 sleep 后触摸、IMU、按键哪些仍可作为唤醒源。

### G. screen off / wake 恢复实验

目标：

- 验证当前模拟器“熄屏后恢复原页面”的假设在真实硬件上的边界。

验收：

- 至少记录一种浅睡眠或屏幕关闭路径。
- 记录唤醒源：按键、触摸、IMU 或定时器。
- 记录 wake 后 RAM、LVGL 对象、页面状态和外设状态是否保持。
- 如果必须重建 UI，记录重建成本和状态保存需求。

不要求：

- 达到最终低功耗指标。
- 实现完整 AOD。
- 接入当前模拟器全部页面。

## 输出物

本轮完成后应产出：

- `docs/twatch_s3_plus_reference_validation_run.md`
- 记录每组实验的状态：未开始 / 通过 / 失败 / 部分通过 / 暂停。
- 记录 BMA423 计步中断异常的复现情况和影响范围。
- 明确哪些问题属于 T-Watch 板级限制，哪些问题会影响后续分立器件选型。

## 进入下一轮的门槛

满足以下条件后，才进入代码 bring-up 或分立器件选型表：

- 明确 T-Watch S3 Plus 的具体版本、LoRa 版本和可用开发环境。
- 明确最小 LVGL 是否能运行。
- 明确触摸、物理按键、BMA423、AXP2101 中哪些路径可用。
- 明确 BMA423 计步中断问题是否只影响计步，还是影响更广泛的中断能力。
- 明确 screen off / wake 的最低可验证路径。
