# v0.8d-1 AppBloodOxygen 首页与设置壳层

日期：2026-05-24

## 目标

把 `AppBloodOxygen` 从 placeholder 收成第一轮真实可达页，但只做页面结构、设置边界和最小共享设置消费，不提前做真实血氧数据源和说明正文。

## 本轮范围

1. `AppBloodOxygen` 首页不再是 placeholder
2. 首页包含：
   - 今日血氧饱和度线图占位
   - 当前血氧百分比占位
   - 上次更新时间占位
   - `开始测量` 按钮视觉壳层
   - `设置`
   - `说明`
3. `血氧设置` 页包含：
   - `全天血氧监测` 开关
   - `低血氧提醒` 入口
4. `低血氧提醒` 页包含：
   - `不提醒 / 90% / 85% / 80%` 四个选项
5. `血氧说明` 本轮只保留壳层，不补具体正文

## 共享模型边界

本轮继续复用 `HealthMonitoringSettingsModel`，并在其中补入：

- `all_day_blood_oxygen_enabled`
- `low_blood_oxygen_reminder_mode`

其中：

- `全天血氧监测` 需要与 `设置 -> 电池 -> 续航优化` 里的同名开关保持一致
- `低血氧提醒` 当前只有 `AppBloodOxygen` 消费，不要求在别处出现

## 明确不做

- 不接真实血氧测量数据源
- 不接真实“开始测量”流程
- 不做说明正文
- 不做低血氧提醒通知链路
- 不做与睡眠、心率、压力页的进一步联动

## 验收标准

1. `AppBloodOxygen` 从启动器或健康入口进入时，不再是 placeholder
2. 首页能看到图表、百分比、更新时间、开始测量按钮、设置、说明
3. `血氧设置` 可达
4. `全天血氧监测` 可切换，并与 `续航优化` 保持一致
5. `低血氧提醒` 可设置为 `不提醒 / 90% / 85% / 80%`
6. 返回 `血氧设置` 后状态文案同步变化
7. `血氧说明` 可达，但正文明确留到下一轮
