# v0.7c-2b-1 通知单条右滑删除闭环

日期：2026-05-24

## 目标

本轮只做一件事：

- 在 `Notifications` 列表态支持单条通知右滑删除

本轮不做：

- 删除背景文案
- 撤销按钮
- 更复杂的阻尼或震动反馈
- 详情页独立化

## 为什么继续这样拆

通知卡现在同时承载三种交互：

- 点按进入详情
- 列表纵向滚动
- 横向拖拽删除

如果一轮把删除背景、撤销、复杂动画和详情联动都一起做上去，
最容易出现的不是“功能没做够”，而是三种输入互相打架。

所以本轮只先收最小可验收版本：

- 右滑跟手
- 不足阈值回弹
- 超过阈值删除

## 当前前后对比

### 改之前

```text
通知卡只能：
- 点击进入详情
- 随列表上下滚动
```

### 改之后

```text
通知卡现在可以：
- 点击进入详情
- 随列表上下滚动
- 向右跟手拖动
- 超过阈值后删除当前这条
```

## 本轮设计决定

### 1. 只支持列表态

单条右滑删除只在 `Notifications` 列表态有效：

- 详情态不支持
- 清空确认层显示时不支持

### 2. 方向只认“向右”

本轮只实现：

- 按住消息向右滑动删除

不支持左滑，也不引入左右不同动作。

### 3. 先保证输入不互相打架

本轮的首要目标不是“动画最华丽”，而是让三种输入边界先讲清楚：

- 轻点仍然进入详情
- 正常上下滑仍然滚动列表
- 明确向右拖过阈值时才进入删除语义

## 修改范围

### 页面层

- `NotificationsPage`
  - 单条卡片增加右滑跟手位移
  - 释放后根据阈值决定删除或回弹
  - 与点击详情、列表滚动做最小冲突隔离

### 保持不变

- `DataCenter::dismiss_notification(...)`
- `NotificationService`
- `NotificationWakePreview`
- `清空全部` 语义

## 验收标准

### O1. 右滑跟手

- 在 `Notifications` 列表态，按住一条通知向右拖
- 应看到卡片跟随手势向右位移

### O2. 不足阈值回弹

- 将卡片向右拖动但不超过阈值
- 松手后卡片应回弹，不删除

### O3. 超过阈值删除单条

- 将卡片向右拖动超过阈值
- 松手后只删除当前这条通知

### O4. 点击与纵向滚动仍正常

- 轻点通知卡
- 仍应进入详情
- 正常上下滑动列表
- 列表滚动仍应正常

### O5. 旧路径不坏

- 继续测试 `清空全部`
- 按 `N` 注入通知
- 按 `B` 测低电量通知
- 以上路径都不应被破坏
