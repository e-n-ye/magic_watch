## 2026-05-14: Web 视觉沙盘作为 LVGL 之前的视觉规格中间层

背景：

## 2026-05-13: 流式页面的滚动需要明确物理反馈规则

背景：

- 用户补充说明：在消息页、设置页、应用页等流式页面中，用户既可以通过触屏上下滑动浏览，也可以通过表冠旋转浏览。

决定：

- 将滚动物理反馈作为独立交互规则处理。
- 当前阶段先定义触摸拖动、快速滑动、惯性衰减和表冠滚动的一致性原则。
- 暂不在文档阶段把具体阻尼、加速度、减速度参数写死。

理由：

- 小屏设备对滚动手感非常敏感，手感差会直接破坏产品质感。
- 这部分属于系统交互体验，不应散落在单个页面里临时处理。
- 先抽象出统一规则，后续更容易在 LVGL 模拟器和真实硬件上对齐。

复盘点：

- 样机阶段重点观察：触摸滚动是否足够跟手，惯性是否自然，表冠滚动是否与触摸体验割裂。

- 直接进入 deep sleep 会同时引入唤醒源、电源域、串口复位和 UI 恢复等变量。

决定：

- 第五闭环先只验证 `setBrightness(0)` 和 `setBrightness(160)` 的显示关闭/恢复路径。
- 使用 PMU 短按侧键在运行态切换 screen off/on。
- screen off 后如果触摸仍可被读取，则触摸恢复显示。
- 串口输出 `[bringup-screen]`，记录状态、原因、次数和亮度。
- 暂不进入 light sleep/deep sleep，暂不验证真实低功耗唤醒源。

理由：

- 先确认“黑屏”和“亮回”本身可靠，才能继续评估更深层的 sleep/wake。
- 这个闭环能区分显示/背光控制问题与 ESP32-S3 低功耗唤醒问题。
- T-Watch S3 Plus 仍只是参考板，实验结果只收敛约束，不决定最终平台。

复盘点：

- 2026-05-22 实物观察确认：短按侧键可稳定黑屏，黑屏后再次短按侧键可稳定恢复，黑屏后触摸屏幕也可稳定恢复。
- 若运行态 screen off/on 可靠，下一轮再拆 light sleep 或 deep sleep 唤醒源实验。
