## 视觉特征

从小米手表参考图抽取出的共性如下：

1. 整屏深色背景，顶部留出清晰的呼吸区。
2. 左上是页面标题，右上是时间，均为高亮白字。
3. 列表由一张张独立深蓝卡片组成，不再放进一个大面板里。
4. 主标题优先，副文案尽量少，只在需要表达当前值时出现。
5. 开关、单选点、亮度滑条使用高亮蓝作为状态色。
6. 亮度滑条页使用纯黑背景，中间竖直滑条，底部只保留简短说明。

## 头部交互约束

- 左上返回箭头只承担“返回上级页面”功能。
- 标题文字本身不承担返回功能。
- 左边缘右滑返回继续保留。
- 表冠按下回表盘的全局规则保持不变。

## 头部点击豁免区

模拟器已经确认存在一条输入层规则：

- 左边缘返回手势的判定区域会覆盖左上角返回箭头。
- 如果不做额外处理，点击返回箭头时容易被误判成边缘右滑候选。

因此，模拟器输入层保留了一个左上角“头部点击豁免区”：

- 该区域内点击优先按普通点击处理。
- 该规则的目的，是保证返回箭头可以稳定点击。
- 该规则不意味着整段标题都应该变成返回按钮。

后续新增类似二级设置页时，优先复用这条输入层规则，而不是继续扩大标题点击区。

## 抬腕亮屏当前约束

当前 `显示与亮度 -> 抬腕亮屏` 采用两轮拆分：

### 已完成的小闭环

- 页面为三态：
  - 关闭
  - 全天开启
  - 定时开启
- `定时开启` 当前只展示固定时段：
  - 开始 `08:00`
  - 结束 `22:00`
- 首页摘要会同步显示：
  - 关闭
  - 全天开启
  - 定时开启

### 模拟器调试键约定

- `R`：模拟“抬腕查看”，对应 `SimRaiseToWake`
- `F`：模拟“离腕/不再查看”，对应 `SimRaiseDismiss`

当前 `R/F` 只用于模拟器验证输入链，后续真实硬件阶段应由传感器消息接管。

### 当前行为边界

- `R` 唤醒是否生效，取决于当前抬腕亮屏模式。
- `定时开启` 只在当前时间落入 `08:00-22:00` 时允许 `R` 唤醒。
- `F` 只结束由抬腕唤醒出来的查看会话，并触发熄屏。
- `开始时间 / 结束时间` 编辑页属于显式确认保存：
  - 按勾：保存并返回上级页
  - 左边缘右滑、左上角返回、表冠返回：不保存

## 当前实现约束

- 优先用纯色块、圆角、字重和间距还原气质。
- 图标优先使用 `LVGL` 内置 symbol + 彩色圆底。
- 不因“更像真机”就提前引入超出当前轮次的高风险策略联动。
- 当任务同时碰到页面层级、模型、输入事件和状态机时，继续按“小闭环”拆分推进。

这条路线只服务于视觉验证，不替代当前模拟器代码主线。

## 持续亮屏当前约束

当前 `显示与亮度 -> 持续亮屏` 继续按“小闭环”推进，先只落下这一轮需要的最小真实行为：

- 页面为二级单选页：
  - 关闭
  - 5分钟
  - 10分钟
  - 15分钟
  - 20分钟
- 选择 `关闭` 时，直接保存并停留当前页。
- 选择非关闭项时，先弹出续航确认层：
  - 左侧 `X`：取消，不保存
  - 右侧 `勾`：确认保存
- 确认文案表达的是“仅本次亮屏持续所选时长”，而不是永久修改全局息屏策略。

### 当前行为边界

- 持续亮屏生效期间：
  - 自动熄屏计时器暂停
  - `R` 触发的抬腕查看结束后，`F` 不会直接熄屏
- 到达所选时长后：
  - 本次持续亮屏自动结束
  - 策略值回到 `关闭`
  - 屏幕进入熄屏
- 如果用户在持续亮屏期间手动触发熄屏或关机：
  - 本次持续亮屏也会被清空
  - 下次重新亮屏后不会继续沿用上一次的时长

### 暂不在本轮实现

- 不与 `息屏显示(AOD)` 共用同一字段语义
- 不加入剩余时间倒计时 UI
- 不做“持续亮屏结束前提醒”
- 不扩展到快捷设置里的独立时长控制

## 快捷设置与持续亮屏

当前快捷设置右下角按钮继续保留“5分钟快捷开启”的语义，但已经从旧的 `AOD` 占位语义切换为真正跟随 `持续亮屏`：

- 点按右下角按钮：
  - 关闭时，直接开启 `5分钟` 持续亮屏
  - 已开启任意持续亮屏时长时，直接关闭
- 长按右下角按钮：
  - 进入 `显示与亮度 -> 持续亮屏`
- 该按钮的高亮状态，按 `keep_screen_on_duration_ms > 0` 判定，而不是按旧 `always_on_display_enabled`

### 当前图标校准

为了更靠近当前小米参考图，快捷设置里做了这一轮的轻量图标校准：

- 左上：`勿扰模式` 使用月牙图标
- 右上：`抬腕亮屏` 使用手表图标
- 右下：`持续亮屏` 使用亮度图标

这轮只收“语义同步 + 最显眼三颗图标”的闭环，不顺手改整套快捷设置功能分布。

### 当前交互补充

- 在快捷设置里直接点按切换：
  - `抬腕亮屏` 显示 `已开启 / 已关闭` 提示气泡
  - `持续亮屏` 显示 `持续亮屏5分钟 / 持续亮屏已关闭` 提示气泡
- 提示气泡位于快捷设置顶栏下方中部，保持单行宽度，短时悬浮，持续约 `3s` 后自动消失。
- 长按快捷设置按钮进入详情页时，不应把“松手”落到目标详情页上：
  - 长按达到阈值后立即跳转
  - 目标详情页入口保留一个很短的首击抑制窗口
  - 用来吸收“长按后松手”带来的误触
- 当前这条防误触规则重点覆盖：
  - `抬腕亮屏`
  - `持续亮屏`
