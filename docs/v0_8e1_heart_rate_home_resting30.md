# v0.8e-1 AppHeartRate 主页面与近30天静息

日期：2026-05-25

## 本轮目标

把 `AppHeartRate` 从 placeholder 推进成第一轮真实可达页。

本轮只做：

- `AppHeartRate` 主页面
- 进入页面后的默认一次 mock 测量
- 测量完成后的当前心率结果展示
- 未佩戴提示弹层
- `近30天静息` 页面

## 为什么拆成三轮

心率线同时涉及主页面、近30天静息、心率设置、心率说明、未佩戴判定、真实测量流程和后续可能的异常提醒。

如果一轮做完，会同时碰页面层级、状态机、共享设置、说明正文和未来数据模型，容易把当前健康线重新扩成大闭环。

因此拆成：

1. `v0.8e-1`：主页面 + 近30天静息
2. `v0.8e-2`：心率设置
3. `v0.8e-3`：心率说明

## 本轮范围

### 1. 主页面

- 标题显示 `心率`
- 进入 app 后默认开始一次 mock 测量
- 测量态显示：
  - 心形视觉
  - `正在测量`
  - `请保持静止`
- 测量完成后显示：
  - 今日心率图表占位
  - `未佩戴`
  - `76 次/分`
  - `76次/分(1分钟前)`

### 2. 未佩戴提示

- 本轮用固定 mock 结果模拟“测量到未佩戴”
- 测量完成后弹出提示：
  - `1.5cm-2cm`
  - `请正确佩戴手表后重试`
  - `知道了`
- 点击 `知道了` 后回到心率结果页

### 3. 主页面入口列表

主页面底部显示三项：

- `近30天静息`
- `心率设置`
- `心率说明`

其中本轮只有 `近30天静息` 进入真实页面。

`心率设置` 和 `心率说明` 只作为后续两轮的可见入口占位，本轮不注册真实页面、不做设置或说明正文。

### 4. 近30天静息

- 标题显示 `近30天静息`
- 页面显示 30 天静息心率折线占位
- 显示日期刻度 `4/26 / 5/10 / 5/25`
- 显示 `-- 次/分` 与 `今日静息`
- 支持触摸滚动和表冠滚动

## 本轮明确不做

- 不接真实心率传感器
- 不实现真实测量流程
- 不实现未佩戴检测算法
- 不新增心率数据模型
- 不做心率异常提醒通知链路
- 不做 `心率设置`
- 不做 `心率说明`
- 不改 `AppStress` / `AppBreathing`

## 验收标准

1. 从 Launcher 或健康快捷面进入 `AppHeartRate`
2. 页面不再是 placeholder
3. 进入页面后先显示 `正在测量 / 请保持静止`
4. 测量完成后弹出未佩戴提示
5. 点击 `知道了` 后显示当前心率结果页
6. 主页面能看到 `近30天静息 / 心率设置 / 心率说明`
7. 点击 `近30天静息` 进入对应页面
8. `近30天静息` 页面图表和 `-- 次/分 / 今日静息` 文案可见
9. 返回、触摸滚动、表冠滚动正常
