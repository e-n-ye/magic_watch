# v0.8e-2 AppHeartRate 心率设置

日期：2026-05-25

## 本轮目标

在 `v0.8e-1` 已完成心率主页面与 `近30天静息` 的基础上，只补 `心率设置` 路径。

本轮不做 `心率说明` 正文，也不扩展真实心率测量、异常通知或未佩戴算法。

## 本轮范围

### 1. 设置入口

- `AppHeartRate` 主页面中的 `心率设置` 从占位入口变为真实可点击入口
- 点击后进入 `心率设置`
- `心率说明` 仍保留为第三轮入口占位

### 2. 心率设置页

`心率设置` 页包含四项：

- `全天心率监测`
- `心脏健康监测`
- `高心率提醒`
- `低心率提醒`

每项显示当前状态，进入子页修改后，返回设置页可看到状态同步更新。

### 3. 全天心率监测

独立选择页 `全天心率监测` 支持：

- `关闭`
- `智能监测`
- `1分钟一次`
- `10分钟一次`
- `30分钟一次`

默认值为 `智能监测`。

本轮只保存选择结果，不接真实后台采样频率。

### 4. 心脏健康监测

独立详情页 `心脏健康监测` 使用 `HealthMonitoringSettingsModel::heart_health_monitoring_enabled`。

这意味着：

- 从 `心率设置` 修改开关，会同步到续航优化页
- 从续航优化页修改 `心脏健康监测`，回到 `心率设置` 也会同步显示

### 5. 高心率提醒

独立选择页 `高心率提醒` 支持：

- `关闭`
- `100次/分`
- `110次/分`
- `120次/分`
- `130次/分`
- `140次/分`
- `150次/分`

默认值为 `关闭`。

本轮只保存阈值，不触发真实提醒通知。

### 6. 低心率提醒

独立选择页 `低心率提醒` 支持：

- `关闭`
- `40次/分`
- `45次/分`
- `50次/分`

默认值为 `关闭`。

本轮只保存阈值，不触发真实提醒通知。

## 代码边界

- `HealthMonitoringSettingsModel` 新增心率全天监测、高心率提醒、低心率提醒配置字段
- `DataCenter` 新增对应 setter，并继续通过 `HealthMonitoringSettingsChanged` 发布变更
- 页面 ID 新增：
  - `AppHeartRateSettings`
  - `AppHeartRateAllDayMonitoring`
  - `AppHeartRateHeartHealthMonitoring`
  - `AppHeartRateHighReminder`
  - `AppHeartRateLowReminder`
- `AppHeartRate` 只把 `心率设置` 接到真实页面
- `心率说明` 不在本轮注册真实页面

## 本轮明确不做

- 不接真实心率采样频率
- 不实现高低心率通知链路
- 不实现未佩戴检测算法
- 不做 `心率说明`
- 不改 `AppStress` / `AppBreathing`

## 验收标准

1. 从 `AppHeartRate` 进入 `心率设置`
2. `心率设置` 显示四项设置及当前状态
3. `全天心率监测` 可选择 `关闭 / 智能监测 / 1分钟一次 / 10分钟一次 / 30分钟一次`
4. 返回设置页后，`全天心率监测` 状态文案立即更新
5. `心脏健康监测` 可开关，并与续航优化页同一共享状态同步
6. `高心率提醒` 可选择 `关闭 / 100 / 110 / 120 / 130 / 140 / 150 次/分`
7. `低心率提醒` 可选择 `关闭 / 40 / 45 / 50 次/分`
8. 各子页返回、触摸滚动、表冠滚动正常
